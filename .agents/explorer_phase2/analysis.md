# AROH Phase 2 Codebase Exploration & Analysis Report

This report documents the architectural, data flow, and code quality findings from exploring the AROH Phase 2 codebase.

---

## 1. Authentication & Session Management

### Centralized Zustand Store
- **Store Location**: `packages/asdk/src/store/index.ts`
- **Hook**: `usePlatformStore`
- **State Properties Managed**:
  - `user`: Holds basic details (`id`, `email`, `role`, `createdAt`).
  - `profile`: Tracks UI/membership status (`userId`, `displayName`, `avatarUrl`, `membershipLevel`, `updatedAt`).
  - `wallet`: Centralizes user token balance (`userId`, `balance`, `updatedAt`).
  - `token`: Cryptographically signed JSON Web Token (production Firebase ID token or a simulated mock token).
  - `isAuthenticated`: Boolean flag representing login status.
  - `isLoading`: Async action status tracker.
  - `error`: Stores API or service failure messages.

### Core Authentication Actions
- **Login (`login(email, password)`)**:
  - Invokes `mockAuthService.login(email, password)`.
  - Sets `user`, `profile`, `wallet`, `token` (cryptographic token), and `isAuthenticated: true`.
  - Pre-fetches transactions via `fetchUserTransactions()`.
  - Emits a success notification.
- **Register (`register(email, displayName, password)`)**:
  - Invokes `mockAuthService.register(email, displayName, password)`.
  - Seeds the account with a starting balance of **500 Aros** and writes a corresponding ledger transaction entry.
  - Sets state parameters and flips `isAuthenticated` to true.
- **Logout (`logout()`)**:
  - Clears all session states (`user`, `profile`, `wallet`, `token`, `isAuthenticated`, `transactions`, `notifications`) back to defaults.

### Dual-Environment Service Infrastructure (`packages/asdk/src/services/firebase.ts`)
- **Environment Detection**: Checks for a `NEXT_PUBLIC_FIREBASE_API_KEY` or `NEXT_PUBLIC_AROH_ENV === "mock"`.
- **Production Mode**:
  - Interfaces directly with Firebase Authentication SDK (`signInWithEmailAndPassword`, `createUserWithEmailAndPassword`).
  - Stores/resolves extended entity profiles and balances in Firestore collections: `users`, `profiles`, `wallets`, and `transactions`.
- **Mock Mode**:
  - Operates entirely in memory and synchronizes state to `localStorage` using keys:
    - `aroh_mock_users` (Seeded with `admin@aroh.co`/`admin`, `operator@aroh.co`/`operator`, `user@aroh.co`/`user`).
    - `aroh_mock_profiles` (Associates profiles to users, defining default roles and membership tiers).
    - `aroh_mock_wallets` (Seeds balances: Admin with 5000 Aros, Operator with 1000 Aros, User with 500 Aros).
    - `aroh_mock_transactions` (Audited history logs).
- **Cryptographic Token Verification (`packages/asdk/src/services/token.ts`)**:
  - Mock mode implements a zero-dependency JWT-like mechanism using base64 header/payload encoding and a `simpleHash` signature generated with a secret key.
  - The token is verified on-demand for expiration and authority inside Next.js route endpoints.

## 2. CMS Announcements

### Configuration & Controls
- **CMS Interface**: Found at `apps/web/app/cms/page.tsx`.
- **RBAC Rules**: clearance is limited to users holding the role `admin` or `operator` (`user?.role === "admin" || user?.role === "operator"`). Other roles get access-denied warnings.
- **CMS Actions**:
  - **Upserting announcements**: Form allows configuring a Title, Content, Category (enum: `info`, `promotion`, `maintenance`), and Status (Published/Draft).
  - Triggers the store action `upsertAnnouncement(announcement)`, calling `mockCmsService.upsertAnnouncement`.
  - **Deleting announcements**: Triggers `deleteAnnouncement(id)`, deleting the document in storage.

### Firestore Collections Involved
- **Collection Name**: `cms` (for production) / local storage key `aroh_mock_cms` (for mock).
- **Document Schema**:
  - `id`: string (UUID or autogenerated key).
  - `title`: string.
  - `content`: string.
  - `category`: `"info" | "promotion" | "maintenance"`.
  - `isPublished`: boolean.
  - `publishedAt`: string (ISO date string).
  - `authorId`: string.

### Rendering on Homepage Hub
- **Location**: Homepage (`apps/web/app/page.tsx`).
- **Data Flow**:
  - Triggers `fetchAnnouncements()` inside a `useEffect` on mount.
  - In production mode, queries the `cms` collection on Firestore filtered by `isPublished == true`, sorted by `publishedAt` descending.
  - Renders the list in the "Ecosystem Announcements" grid, styled with dynamic visual badges for each category.

## 3. Ecosystem Search Mechanisms

### Explore Hub Page (`apps/web/app/explore/page.tsx`)
- **Product Registry Data**: A static array of `ProductDetails` (`registeredProducts`) is hardcoded in the file.
- **Registered Apps**:
  1. `aros-wallet`: Aros Core Wallet (Basic tier required, 0 Aros).
  2. `aroh-cms`: Aroh CMS Alerts (Pro tier required, 100 Aros).
  3. `aros-metrics`: Aros Metrics Engine (Pro tier required, 100 Aros).
  4. `aroh-notifier`: Aroh Notification Center (Pro tier required, 100 Aros).
  5. `aros-console`: Aros Command Console (Enterprise tier required, 500 Aros).
  6. `aroh-ai-helper`: AROH AI Doc Helper (Enterprise tier required, 500 Aros).
- **Search Implementation**: Performs client-side search against the query state (`searchQuery`) matching `name` or `description` case-insensitively.
- **Category Filtering**: Filters products locally matching the selected category button against the `badge` field.

### Command Palette Component (`apps/web/app/components/command-palette.tsx`)
- **Trigger**: Globally accessible inside the root layout via `Ctrl + K` or `Cmd + K`.
- **Item Indexing**:
  - Builds and memoizes a list of shortcut commands based on `isAuthenticated` and `user?.role`.
  - Combines:
    - **Navigation paths** (`/`, `/dashboard`, `/explore`, `/ai`).
    - **Admin tools** (`/admin` for admins, `/cms` for admins/operators).
    - **Products**: Imports the static `registeredProducts` from the explore page, wrapping each as a product item.
    - **Actions** (Sign Out).
  - **Search Implementation**: A local input query state `search` filters the memoized command items list based on matching item `name` or `category` strings.

### AI Hub Principle Search (`apps/web/app/ai/page.tsx`)
- Contains a localized documentation indexing helper.
- Standardizes principles under a mock database array `mockDocDatabase` (indexing keywords like `"ledger"`, `"monorepo"`, `"single source of truth"`, `"backend authority"`).
- Query matches entries matching the input query string locally.

## 4. Metrics Dashboard

### Admin Page Status (`apps/web/app/admin/page.tsx`)
- An admin console page exists at `apps/web/app/admin/page.tsx`.
- However, its functionality is limited to **issuing token incentives** (`rewardUser` store action) and inspecting the **global transaction ledger** (`getAllTransactions()` from SDK).

### Recharts Dependency Analysis
- **Package Status**: `recharts` is present in `apps/web/package.json` under dependencies (`"recharts": "^2.12.0"`).
- **Code Usage**: A search across the codebase confirms `recharts` is **never imported** or configured in any component.
- **Metrics Dashboard Status**: The Aros Metrics Engine (`aros-metrics`) product details page only displays a mock terminal log simulation on launch. No visual metrics dashboard exists yet.
- **Downstream Strategy**: Implementing a dashboard requires creating a new metrics page (e.g. at `/admin/metrics` or under the metrics app) importing recharts elements (`ResponsiveContainer`, `AreaChart`, `XAxis`, etc.) and feeding them with real-time stats (e.g., wallet volume, API latency) or mock telemetry.

## 5. Code Quality, Layout, and Testing

### Technology Stack & Build Tools
- **Monorepo Strategy**: Managed via `npm workspaces` splitting directories into `apps` and `packages`.
- **Framework**: Next.js 16 (`16.2.10`) with React 19 (`19.2.4`) app router.
- **CSS System**: Tailwind CSS v4 using PostCSS.
- **Language**: TypeScript 5.
- **State & Schemas**: Zustand 5 and Zod 3.

### Directory Layout
```
AROH/
├── apps/
│   └── web/                   # Next.js App Router workspace (pages, components, public assets)
├── packages/
│   ├── asdk/                  # central AROH Software Development Kit
│   │   └── src/
│   │       ├── schemas/       # Zod schemas & TypeScript types
│   │       ├── services/      # Firebase clients (Mock & Production) and JWT tokens
│   │       ├── store/         # Unified Zustand state store
│   │       └── index.ts       # SDK exports
│   └── ads/                   # AROH Design System components (Button, utility configurations)
└── scripts/                   # Workspace execution scripts
```

### Automated Testing
- **Test File**: `scripts/test-sdk.js`
- **Testing Approach**: Pure Node.js environment simulation.
  - Stubs global `localStorage` inside Node.js to verify state mutation logic.
  - Imports services directly from `packages/asdk/src/services/firebase.ts`.
- **Test Coverage**:
  1. **Authentication**: Resolves login checks and confirms default seeded admin & user balances.
  2. **Wallet Credits**: Validates crediting user wallets and logs ledger entries.
  3. **Membership Upgrades**: Verifies balance deductions and upgrade profile updates.
  4. **Fraud Checks**: Asserts that trying to upgrade with an insufficient balance throws an error and reverts the ledger.
- **Test Command**: `node scripts/test-sdk.js`
