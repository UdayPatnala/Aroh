# Purpose

This document defines how AI prompts shall be used throughout the AROH project.

It establishes prompt hierarchy, execution order, responsibilities, continuity rules, and collaboration standards between specialized AI engineering roles.

This document governs every future AI-assisted engineering session.

---

# Objective

Ensure every AI coding session produces:

- Consistent Engineering Decisions
- Predictable Outputs
- Architecture Compliance
- Documentation Synchronization
- Production-Ready Implementations

AI should function as a coordinated engineering organization rather than isolated assistants.

---

# Prompt Execution Order

Every engineering session shall follow this order.

1. Read Engineering Contract

2. Read AI Context

3. Read Phase 1 Scope

4. Read Development Standards

5. Read Repository Guide

6. Read Architecture Summary

7. Execute Master Prompt

8. Execute Specialized Prompt

9. Perform Implementation

10. Update Documentation

11. Perform Validation

# Prompt Responsibilities

## Master Prompt

Defines:

Project Context

Engineering Rules

Decision Framework

Execution Strategy

Quality Gates

Autonomous Behavior

Every specialized prompt inherits these rules.

---

## Specialized Prompts

Each specialized prompt is responsible only for its engineering discipline.

Examples include:

Frontend

Backend

Database

Security

Performance

QA

Accessibility

Documentation

Refactoring

Architecture Review

Production Validation

Specialized prompts must not redefine project-wide rules.

---

# Context Preservation

Before making changes, AI shall understand:

Current Architecture

Existing Implementation

Current Phase

Previous Decisions

Applicable Standards

AI shall preserve established project conventions throughout the session.

---

# Session Continuity

Implementation should continue until:

Assigned work is complete.

Critical project information is unavailable.

Human approval is explicitly required.

Do not stop after every small task.

Stop only at meaningful milestones.

---

# Decision Policy

Engineering decisions shall be made autonomously whenever sufficient information exists.

Avoid unnecessary clarification requests.

If multiple valid solutions exist, select the objectively strongest solution according to the Engineering Contract.

---

# Documentation Synchronization

Whenever implementation changes:

Update affected documentation.

Maintain consistency across documents.

Avoid documentation drift.

Never duplicate documentation.

Reference authoritative documents whenever possible.

---

# Scope Enforcement

AI shall implement only functionality approved for the current phase.

If future functionality is encountered:

Prepare extension points where appropriate.

Do not implement future features.

Prevent feature creep.

---

# Quality Expectations

Every AI-generated implementation shall satisfy:

Architecture

Security

Accessibility

Performance

Maintainability

Reliability

Scalability

Testing

Documentation

Production Readiness

Implementation is incomplete until applicable quality gates pass.

# Code Review Expectations

Before considering work complete, AI shall verify:

No duplicated logic.

No dead code.

No unused dependencies.

No architectural violations.

No security regressions.

No accessibility regressions.

No performance regressions.

Documentation is synchronized.

Architecture Summary

Changes to prompt behavior shall not contradict immutable bootstrap documents.
